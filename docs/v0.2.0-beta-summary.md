## What's New

### ðŸš€ Major Features

#### 1. Compound AI Support
Multi-model orchestration for sophisticated AI workflows:
- Custom model selection (answering + reasoning models)
- Tool configuration (web search, code interpreter, Wolfram)
- Executed tool results tracking (browser, code, search)

**Example:**
```go
resp, err := client.Chat.Create(ctx, &types.CreateChatCompletionRequest{
    Model: string(types.ModelCompoundBeta),
    Messages: messages,
    CompoundCustom: &types.CompoundCustom{
        Models: &types.CompoundCustomModels{
            AnsweringModel: option.Ptr(option.Some("llama-3.3-70b-versatile")),
            ReasoningModel: option.Ptr(option.Some("qwen/qwen3-32b")),
        },
        Tools: &types.CompoundCustomTools{
            EnabledTools: []string{"web_search"},
        },
    },
})
```

#### 2. Documents & Citations (RAG)
Provide context documents with automatic citation tracking:
- Text and JSON document sources
- Document and function call citations
- Citation options control

**Example:**
```go
resp, err := client.Chat.Create(ctx, &types.CreateChatCompletionRequest{
    Model: "llama-3.3-70b-versatile",
    Messages: messages,
    Documents: []types.Document{
        {
            ID: &docID,
            Source: &types.DocumentSource{
                Type: "text",
                Text: &documentContent,
            },
        },
    },
    CitationOptions: option.Ptr(option.Some("enabled")),
})
```

#### 3. Reasoning Models
Advanced reasoning with configurable output:
- Reasoning effort levels (none, low, medium, high)
- Reasoning format options (hidden, raw, parsed)
- Reasoning content in responses

**Example:**
```go
resp, err := client.Chat.Create(ctx, &types.CreateChatCompletionRequest{
    Model: string(types.ModelQwen332B),
    Messages: messages,
    ReasoningEffort:  option.Ptr(option.Some("medium")),
    ReasoningFormat:  option.Ptr(option.Some("parsed")),
    IncludeReasoning: option.Ptr(option.Some(true)),
})
```

#### 4. Enhanced Streaming Metadata
Rich metadata in streaming responses via `XGroq` field:
- Request ID tracking
- Debug information (token IDs and strings)
- Per-model usage breakdown for compound AI
- Error reporting in streams

**Example:**
```go
for {
    chunk, err := stream.Next(ctx)
    if chunk.XGroq != nil && chunk.XGroq.Usage != nil {
        fmt.Printf("Usage: %+v\n", chunk.XGroq.Usage)
        if chunk.XGroq.UsageBreakdown != nil {
            fmt.Printf("Per-model breakdown: %+v\n", chunk.XGroq.UsageBreakdown)
        }
    }
}
```

## Migration Guide

### From v0.1.0 to v0.2.0-beta

**No migration needed!** All existing code works as-is.

To use new features, simply add the new fields to your requests:

```go
// Before (v0.1.0) - still works
resp, err := client.Chat.Create(ctx, &types.CreateChatCompletionRequest{
    Model: "llama-3.1-8b-instant",
    Messages: messages,
})

// After (v0.2.0-beta) - with new features
resp, err := client.Chat.Create(ctx, &types.CreateChatCompletionRequest{
    Model: string(types.ModelCompoundBeta),
    Messages: messages,
    CompoundCustom: &types.CompoundCustom{...},  // New!
    Documents: documents,                         // New!
    SearchSettings: &types.SearchSettings{...},   // New!
})
```
